银行业可疑交易报告核心字段分类及说明

银行业可疑交易报告核心字段分类及说明
一、基础信息类（一级特征）—— 数据来源：机构基础信息系统 / 综合管理岗
此类字段聚焦报告机构、网点及报告本身的基础标识，由综合管理岗或反洗钱专职岗从机构内部基础信息系统提取录入，是报告的 “身份信息”。
XML 标签	字段内容	说明
RICD	报告机构编码	字符型（14 位），为人民银行发放的《金融机构代码证》上载明的金融机构编码，无代码时由中国反洗钱监测分析中心分配，从机构资质档案或基础信息系统中调取。
RPNC	上报网点代码	字符型（16 位），为发现可疑交易的金融机构网点唯一代码，有金融机构代码的用官方代码，无代码的由机构自行编制并报备，从网点信息管理系统提取。
RPNM	可疑交易报告填报人员	字符型（16 位），填写实际填报报告的人员姓名或工号，由填报岗直接录入，便于后续责任追溯。
DETR	可疑交易报告紧急程度	字符型（2 位），01 表示 “非特别紧急”，02 表示 “特别紧急”，由反洗钱岗根据交易风险等级判断选择，高风险交易（如涉嫌恐怖融资）选 02。
TORP	报送次数标志	字符型（5 位），初次报送填 1，后续补报或纠错逐次累加，超过 9999 填 0000，由系统自动计数或填报岗手动维护。
ORXN	初次报送的可疑交易报告报文名称	字符型（64 位），非初次报送时填写首次报告的报文名称（不加扩展名），初次报送填 “@N”，从历史报送记录中查询录入。
DORP	报送方向	字符型（2 位），01 为 “仅报告中国反洗钱监测分析中心”，02 为 “报告中心 + 人民银行当地分支机构”，03 为 “报告中心 + 当地公安机关”，04 为 “报告中心 + 分支机构 + 公安机关”，99 为 “报告中心 + 其他机构”，由反洗钱岗根据监管要求选择。
ODRP	其他报送方向	字符型（32 位），仅当 DORP 为 99 时填写具体 “其他机构” 名称（如银保监会），否则填 “@N”，由填报岗根据实际报送对象补充。
二、可疑主体信息类（一级特征）—— 数据来源：客户信息管理系统 / 客户经理岗
此类字段围绕可疑交易相关主体（客户、控股股东、法定代表人等）展开，由客户经理岗从客户信息管理系统提取，缺失信息需联系客户补充，是识别主体身份的核心。
XML 标签	字段内容	说明
CSNM	客户号	字符型（32 位），为报告机构内部唯一标识客户的编号，从核心业务系统的客户档案中直接调取，确保与客户账户信息关联。
CTNM	客户姓名 / 名称	字符型（512 位），个人客户填身份证姓名，机构客户填营业执照上的全称，与客户身份证件或登记文件一致，从客户信息系统提取。
CITP	客户身份证件 / 证明文件类型	字符型（6 位），按 10.1 节《身份证件 / 证明文件类型代码表》填写（如 110001 为居民身份证、610001 为全国组织机构代码），从客户开户资料中调取。
CTID	客户身份证件 / 证明文件号码	字符型（128 位），居民身份证填 15 位或 18 位号码，组织机构代码填 9 位（去除 “-”），与证件原件一致，从客户身份认证系统提取。
SEVC	可疑主体职业（对私）或行业（对公）	字符型（32 位），对私客户按 GB/T 6565-2015《职业分类与代码》填 “大类 / 中类 / 小类”（如 “08 - 专业技术人员”），对公客户按 GB/T 4754-2011《国民经济行业分类》填 “门类 / 大类”（如 “F - 批发和零售业”），失业 / 无业填 “99999”，从客户尽职调查资料中提取。
STNT	可疑主体国籍	字符型（3 位），按 GB/T 2659-2000《世界各国和地区名称代码》填写（如 CHN 为中国、USA 为美国），多国籍需分别填写，从客户身份信息中调取。
SCTL	可疑主体联系电话	字符型（64 位），手机填 11 位号码，固定电话填 “区号 - 号码”（如 010-12345678），可填多个，从客户联系方式档案中提取。
SEAR	可疑主体住址 / 经营地址	字符型（512 位），个人客户填常住地址，机构客户填注册经营地址，可填多个，与客户登记资料一致。
SEEI	可疑主体其他联系方式	字符型（512 位），填写电话、地址外的联系方式（如邮箱、微信），有则填，无则填 “@N”，从客户补充资料中提取。
SRNM	可疑主体法定代表人姓名	字符型（512 位），仅机构客户填写，与营业执照或登记文件上的法定代表人姓名一致，从机构客户工商资料中调取。
SRIT	可疑主体法定代表人身份证件类型	字符型（32 位），按 10.1 节代码表填写（如居民身份证、护照），从法定代表人身份资料中提取。
SRID	可疑主体法定代表人身份证件号码	字符型（128 位），与法定代表人身份证件原件一致，居民身份证填 15/18 位，从身份认证资料中提取。
SCNM	可疑主体控股股东或实际控制人名称	字符型（512 位），填写对客户有实际控制权的个人或机构名称，从客户股权结构或尽职调查资料中提取。
SCIT	可疑主体控股股东或实际控制人身份证件类型	字符型（6 位），按 10.1 节代码表填写，个人填居民身份证等，机构填组织机构代码等。
SCID	可疑主体控股股东或实际控制人身份证件号码	字符型（128 位），与对应身份证件一致，确保身份可追溯。
SETN	可疑主体总数	数值型（8 位），填写报告中涉及的所有可疑主体（客户、关联方等）总数量，由填报岗统计录入。
三、账户与交易信息类（一级特征）—— 数据来源：核心业务系统 / 运营岗
此类字段涵盖客户账户属性及交易明细，由运营岗从核心业务系统（如账户管理模块、交易流水模块）提取，是还原交易场景的关键。
XML 标签	字段内容	说明
CATP	客户账户类型	字符型（6 位），按 10.2 节《账户类型代码表》填写（如 120001 为活期储蓄账户、620001 为组织基本账户），从账户开户信息中调取。
CTAC	客户账号	字符型（64 位），为客户在报告机构开立的账户号码（如银行卡号、对公账号），与核心业务系统中的账户编号一致。
OATM	客户账户开立时间	字符型（14 位），格式为 “YYYYMMDDHHMMSS”（如 20231001093000），从账户开户记录中提取。
CATM	客户账户销户时间	字符型（14 位），账户已销户的填销户时间（格式同上），未销户的填 “@N”，从账户状态管理模块提取。
CBCT	客户银行卡类型	字符型（2 位），10 为借记卡、20 为贷记卡、30 为准贷记卡、90 为其他，仅银行卡账户填写，从银行卡开户信息中调取。
OCBT	客户银行卡其他类型	字符型（32 位），仅当 CBCT 为 90 时填写具体类型（如 “预付卡”），否则填 “@N”，从银行卡属性资料中补充。
CBCN	客户银行卡号码	字符型（64 位），仅银行卡账户填写，与银行卡正面号码一致，从银行卡账户信息中提取。
TSTM	交易时间	字符型（14 位），格式为 “YYYYMMDDHHMMSS”，精确到秒，从交易流水记录中调取。
TRCD	交易发生地	字符型（9 位），中国大陆地区填 “CHN+6 位行政区划代码”（如 CHN110101 为北京东城区），境外填 “3 位国别代码 + 000000”（如 USA000000），从交易地域信息中提取。
TSTP	交易方式	字符型（6 位），按 10.3 节《交易方式代码表》填写（如 0030 为现金支付、0020 为网上支付），从交易结算方式记录中调取。
OCTT	非柜台交易方式	字符型（2 位），11 为网上交易、12 为 POS 交易、13 为电话交易、15 为 ATM 交易等，99 为其他，仅非柜台交易填写，从交易渠道信息中提取。
OOCT	其他非柜台交易方式	字符型（32 位），仅当 OCTT 为 99 时填写具体方式（如 “刷脸支付”），否则填 “@N”，从交易渠道补充资料中提取。
OCEC	非柜台交易方式的设备代码	字符型（500 位），网上交易填 “IP@MAC”（如 192.168.1.1@00-11-22-33-44-55），POS 交易填 POS 机编号，ATM 交易填 ATM 编号，从交易设备记录中提取。
TICD	业务标识号	字符型（256 位），为单笔交易的唯一业务编号（如流水号、订单号），确保与交易记录一一对应，从交易明细中调取。
CRTP	交易币种	字符型（3 位），按 GB/T 12406-2008《表示货币和资金的代码》填写（如 CNY 为人民币、USD 为美元），从交易金额记录中提取。
CRAT	交易金额	数值型（20 位），按实际交易币种计价，保留 3 位小数（如 1000.500），从交易流水的金额字段提取。
TSDR	资金收付标志	字符型（2 位），01 为 “收”（资金流入客户账户），02 为 “付”（资金流出客户账户），从交易方向记录中判断。
CRSP	资金来源与用途	字符型（128 位），填写交易资金的来源（如 “工资收入”）或用途（如 “购房”），有明确信息则填，无则填 “@N”，从交易备注或客户说明中提取。
STNM	可疑交易总数	数值型（8 位），填写报告中涉及的可疑交易总笔数，由运营岗统计交易流水后录入。
四、交易对手与关联信息类（一级特征）—— 数据来源：交易对手信息系统 / 清算岗
此类字段聚焦交易对手的身份及关联交易信息，由清算岗从交易对手信息系统或清算记录中提取，用于追溯交易链路。
XML 标签	字段内容	说明
TCNM	交易对手姓名 / 名称	字符型（128 位），个人对手填姓名，机构对手填全称，从交易对手信息记录中提取。
TCIT	交易对手身份证件 / 证明文件类型	字符型（6 位），按 10.1 节代码表填写，与交易对手身份类型匹配，从对手身份资料中调取。
TCID	交易对手身份证件 / 证明文件号码	字符型（128 位），与对手身份证件一致，确保身份可查，从对手身份认证记录中提取。
TCAT	交易对手账户类型	字符型（6 位），按 10.2 节代码表填写，与对手账户属性匹配，从对手账户信息中调取。
TCAC	交易对手账号	字符型（64 位），填写交易对手的账户号码（如银行卡号、对公账号），从交易清算记录中提取。
CFIN	对方金融机构网点名称	字符型（64 位），若交易对手账户在其他金融机构，填写该机构网点全称（如 “中国工商银行北京朝阳支行”），从清算机构信息中调取。
CFIC	对方金融机构网点代码	字符型（16 位），填写对方网点的官方代码（如现代化支付系统行号），无代码的填 “@N”，从金融机构代码库中查询。
CFCT	对方金融机构网点代码类型	字符型（2 位），11 为现代化支付系统行号、12 为人民币结算账户管理系统行号、13 为银行内部机构号、14 为金融机构代码、99 为其他，与 CFIC 类型匹配。
CFRC	对方金融机构网点行政区划代码	字符型（9 位），规则同 “交易发生地”，填写对方网点所在地区的编码，从网点地域信息中提取。
RPMT	收付款方匹配号类型	字符型（2 位），按 10.5 节《收付款方匹配号类型代码表》填写（如 01 为大小额支付系统清算、02 为银联清算），从交易清算方式中判断。
RPMN	收付款方匹配号	字符型（500 位），按对应匹配号类型规则填写（如银联清算填 “交易传输时间 + 系统跟踪号”），从清算匹配记录中提取。
BPTC	银行与支付机构之间的业务交易编码	字符型（500 位），若交易涉及支付机构，填写相关业务的交易编码，从支付业务清算记录中提取。
TSCT	涉外收支交易分类与代码	数值型（16 位），仅涉外交易填写，按 GB/T 19583-2014《涉外收支交易分类与代码》填写，从涉外交易申报记录中提取。
ROTF	交易信息备注	字符型（64 位），暂填写 “@N”，后续若有补充说明可更新，由清算岗根据实际情况维护。
五、可疑分析与标识类（一级特征）—— 数据来源：反洗钱监测系统 / 反洗钱岗
此类字段是对交易可疑性的判断与标识，由反洗钱岗结合监测系统预警、人工分析结果填写，体现报告的核心风险判断。
XML 标签	字段内容	说明
TPTR	可疑交易报告触发点	字符型（2 位），01 为模型筛选、02 为执法部门指令、03 为监管部门指令、04 为机构内部案件、05 为社会舆情、06 为从业人员发现、99 为其他，根据报告触发原因选择。
OTPR	其他可疑交易报告触发点	字符型（2000 位），仅当 TPTR 为 99 时填写具体触发点（如 “客户频繁拆分交易规避大额监测”），否则填 “@N”，由反洗钱岗详细描述。
STCB	资金交易及客户行为情况	字符型（10000 位），详细描述客户的资金交易特征（如 “3 天内累计转账 10 笔，金额共 500 万元”）及异常行为（如 “非营业时间频繁交易”），基于交易流水和行为记录整理。
AOSP	疑点分析	字符型（10000 位），分析交易可疑的原因（如 “交易金额与客户职业收入不符，且对手为高风险地区账户”），结合风险特征和监管要求撰写。
TOSC	疑似涉罪类型	字符型（4 位），按 10.6 节《疑似涉罪类型代码表》填写（如 0101 为毒品犯罪、0301 为恐怖活动犯罪），可填多个，根据交易可疑特征匹配。
STCR	可疑交易特征代码	字符型（32 位），填写报告机构自主定义的可疑交易监测标准代码（需提前报备），可填多个，匹配机构内部风险模型。
MIRS	人工补正标识	字符型（64 位），仅应答人工补正通知时填写补正通知文件名，其他情况填 “@N”，从补正通知文件中提取文件名。
六、代办人信息类（一级特征）—— 数据来源：代办业务记录 / 柜员岗
此类字段仅当交易存在代办人时填写，由柜员岗从代办业务登记记录中提取，无代办人则相关字段填 “@N”。
XML 标签	字段内容	说明
TBNM	交易代办人姓名	字符型（128 位），填写代办人的真实姓名，与代办人身份证件一致，从代办业务登记单中提取。
TBIT	交易代办人身份证件 / 证明文件类型	字符型（6 位），按 10.1 节代码表填写（如居民身份证、护照），从代办人身份资料中调取。
TBID	交易代办人身份证件 / 证明文件号码	字符型（128 位），与代办人身份证件原件一致，居民身份证填 15/18 位，从身份核验记录中提取。
TBNT	交易代办人国籍	字符型（3 位），按 GB/T 2659-2000 国别代码填写，从代办人身份信息中调取。
