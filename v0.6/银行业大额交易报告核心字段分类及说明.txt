银行业大额交易报告核心字段分类及说明
一、报告基础信息类（一级特征）—— 数据来源：反洗钱管理系统 / 反洗钱专职岗
此类字段为报告的 “身份标识”，聚焦报告机构、报告数量等基础属性，由反洗钱专职岗从机构反洗钱管理系统或基础信息库中提取录入，确保报告可追溯、可统计。
XML 标签	字段内容	说明
RICD	报告机构编码	字符型（14 位），为人民银行发放的《金融机构代码证》上载明的金融机构编码，未取得代码的机构由中国反洗钱监测分析中心分配，从机构资质档案或基础信息系统调取，是报告机构的唯一标识。
CTTN	大额交易客户总数	数值型（8 位），填写本报告中涉及的所有大额交易客户数量，需满足 “客户总数≥客户序号”，由反洗钱岗统计报告内客户数量后录入，反映报告覆盖的客户范围。
二、客户基础信息类（一级特征）—— 数据来源：客户信息管理系统 / 客户经理岗
此类字段围绕大额交易客户的核心身份信息展开，由客户经理岗从客户信息管理系统（CIF 系统）提取，缺失信息需通过客户尽职调查补充，是识别客户身份、关联交易的基础。
XML 标签	字段内容	说明
CSNM	客户号	字符型（32 位），为报告机构内部唯一标识客户的编号（如核心系统客户 ID），从客户档案中直接调取，确保与客户账户、交易记录精准关联。
CTNM	客户姓名 / 名称	字符型（512 位），个人客户填写身份证登记的姓名，机构客户填写营业执照上的全称，与客户身份文件一致，从客户基础信息模块提取，避免别名或简称。
CTID	客户身份证件 / 证明文件号码	字符型（128 位），居民身份证填写 15 位或 18 位号码，组织机构代码填写 9 位（去除校验码前的 “-”），其他证件按实际号码填写，从客户身份认证记录中调取，确保真实性。
CITP	客户身份证件 / 证明文件类型	字符型（6 位），按 10.1 节《身份证件 / 证明文件类型代码表》填写（如 110001 为居民身份证、610001 为全国组织机构代码），从客户开户时留存的身份资料中提取，匹配证件实际类型。
CTNT	客户国籍	字符型（3 位），按 GB/T 2659-2000《世界各国和地区名称代码》填写（如 CHN 为中国、USA 为美国），客户有多个国籍时需分别填写多个字段，从客户身份信息中调取。
CTVC	客户职业（对私）或客户行业（对公）	字符型（32 位），对私客户按 GB/T 6565-2015《职业分类与代码》填写 “大类 / 中类 / 小类”（如 “09 - 办事人员和有关人员”），对公客户按 GB/T 4754-2011《国民经济行业分类》填写 “门类 / 大类”（如 “G - 交通运输、仓储和邮政业”），失业 / 无业 / 离退休填 “99999”，从客户尽职调查资料中提取。
CCTL	客户联系电话	字符型（64 位），手机填写 11 位号码（如 13800138000），固定电话填写 “区号 - 号码”（如 021-61234567），客户有多个电话时需分别填写多个字段，从客户联系方式档案中提取。
CTAR	客户住址 / 经营地址	字符型（512 位），个人客户填写常住地址，机构客户填写注册经营地址，客户有多个地址时需分别填写多个字段，与客户登记资料一致，从客户地址信息模块提取。
CCEI	客户其他联系方式	字符型（512 位），填写电话、地址外的联系方式（如邮箱、微信账号），无其他联系方式时填 “@N”，从客户补充信息记录中提取，便于后续联系核实。
三、账户信息类（一级特征）—— 数据来源：核心业务系统 / 运营柜员岗
此类字段聚焦客户在报告机构开立的账户属性，由运营柜员岗从核心业务系统（账户管理模块）提取，直接关联客户交易账户，确保账户信息与交易记录匹配。
XML 标签	字段内容	说明
CATP	客户账户类型	字符型（6 位），按 10.2 节《账户类型代码表》填写（如 120001 为活期储蓄账户、620001 为组织基本账户），从账户开户信息中调取，反映账户核心属性。
CTAC	客户账号	字符型（64 位），为客户在报告机构开立的账户号码（如银行卡号、对公结算账号），与核心业务系统中的账户编号完全一致，从账户明细中提取，确保交易与账户精准绑定。
OATM	客户账户开立时间	字符型（14 位），格式为 “YYYYMMDDHHMMSS”（如 20220510143000），从账户开户记录中提取，反映账户开立时长，辅助判断交易合理性。
CBCT	客户银行卡类型	字符型（2 位），仅银行卡账户填写，10 为借记卡、20 为贷记卡、30 为准贷记卡、90 为其他，从银行卡开户属性中调取，无银行卡账户时不填。
OCBT	客户银行卡其他类型	字符型（32 位），仅当 CBCT 为 90（其他）时填写具体类型（如 “预付卡”“虚拟信用卡”），否则填 “@N”，从银行卡特殊属性记录中补充，完善银行卡信息。
CBCN	客户银行卡号码	字符型（64 位），仅银行卡账户填写，与银行卡正面印制的号码一致（含卡号中的空格或连字符需按实际格式填写），从银行卡账户信息中提取，无银行卡账户时不填。
FINC	金融机构网点代码	字符型（16 位），为客户开户或办理交易的网点代码，有官方金融机构代码的用官方代码，无代码的由机构自行编制并报备，从网点信息管理系统提取，标识交易办理网点。
RLFC	金融机构与客户的关系	字符型（2 位），00 为 “客户通过境内账户 / 银行卡交易（开户机构 / 发卡行）”，01 为 “客户通过境外银行卡交易（收单行）”，02 为 “客户不通过账户 / 银行卡交易（业务办理机构）”，由运营岗根据交易场景判断选择。
四、大额交易核心信息类（一级特征）—— 数据来源：交易流水系统 / 清算岗
此类字段是大额交易的 “核心载体”，记录交易时间、金额、特征等关键信息，由清算岗从核心业务系统的交易流水模块提取，确保交易细节完整、准确。
XML 标签	字段内容	说明
HTDT	大额交易发生日期	字符型（8 位），格式为 “YYYYMMDD”（如 20240315），从交易流水的交易日期字段提取，同一客户当日同一特征的交易需集中报送，是交易时间的核心标识。
CRCD	大额交易特征代码	字符型（4 位），按监管规定填写（0501 为 “当日单笔 / 累计现金交易≥5 万人民币 / 1 万美金”，0502 为 “非自然人账户间单笔 / 累计转账≥200 万人民币 / 20 万美金”，0503 为 “自然人账户间境内转账≥50 万人民币 / 10 万美金”，0504 为 “自然人账户间跨境转账≥20 万人民币 / 1 万美金”），由清算岗根据交易金额和类型匹配代码。
TTNM	交易总数	数值型（8 位），填写符合某一大额交易特征（CRCD）的交易总笔数，从同一特征下的交易流水统计得出，反映该特征覆盖的交易数量。
TSTM	交易时间	字符型（14 位），格式为 “YYYYMMDDHHMMSS”（如 20240315094530），精确到秒，从每笔交易的流水记录中提取，还原交易发生的具体时间。
TRCD	交易发生地	字符型（9 位），中国大陆地区填写 “CHN+6 位行政区划代码”（如 CHN310104 为上海徐汇区），境外填写 “3 位国别代码 + 000000”（如 JPN000000 为日本），从交易地域信息中提取，标识交易发生的地理位置。
TSTP	交易方式	字符型（6 位），按 10.3 节《交易方式代码表》填写（如 0030 为现金支付、0020 为网上支付、0004 为支票），从交易结算方式记录中调取，反映交易的实现方式。
OCTT	非柜台交易方式	字符型（2 位），仅非柜台交易填写，11 为网上交易、12 为 POS 交易、13 为电话交易、15 为 ATM 交易、99 为其他，从交易渠道信息中提取，无非柜台交易时不填。
OOCT	其他非柜台交易方式	字符型（32 位），仅当 OCTT 为 99（其他）时填写具体方式（如 “刷脸支付”“远程柜台交易”），否则填 “@N”，从交易渠道补充记录中提取，完善非柜台交易信息。
OCEC	非柜台交易方式的设备代码	字符型（500 位），网上交易填写 “IP 地址 @MAC 地址”（如 192.168.0.1@00-1A-2B-3C-4D-5E），POS 交易填写 POS 机编号，ATM 交易填写 ATM 编号，从交易设备记录中提取，非柜台交易时必填，柜台交易填 “@N”。
CRTP	交易币种	字符型（3 位），按 GB/T 12406-2008《表示货币和资金的代码》填写（如 CNY 为人民币、USD 为美元、EUR 为欧元），从交易金额的币种字段提取，是金额计价的基础。
CRAT	交易金额	数值型（20 位），按交易实际币种填写，小数点后最多保留 3 位小数（如 15000.000、2000.500），从交易流水的实际金额字段提取，反映交易的原始金额。
CRMB	交易金额（折人民币）	数值型（20 位），将 CRAT 的金额按当日汇率折算为人民币，小数点后最多保留 3 位小数（如 15000.000、13200.800），由系统自动折算或清算岗根据官方汇率计算填写，便于统一统计。
CUSD	交易金额（折美元）	数值型（20 位），将 CRAT 的金额按当日汇率折算为美元，小数点后最多保留 3 位小数（如 2100.000、2800.300），规则同 CRMB，满足跨境交易统计需求。
TSDR	资金收付标志	字符型（2 位），01 为 “收”（资金流入客户账户），02 为 “付”（资金流出客户账户），从交易方向记录中判断，明确资金流动方向。
CRPP	资金用途	字符型（128 位），填写交易资金的用途（如 “工资发放”“货款支付”“购房款”），有明确信息则填，无则填 “@N”，从交易备注或客户说明中提取，辅助判断交易合理性。
TICD	业务标识号	字符型（256 位），为单笔交易的唯一业务编号（如流水号、订单号、凭证号），从交易明细中调取，确保每笔交易可单独追溯，是交易的 “唯一身份证”。
五、交易对手信息类（一级特征）—— 数据来源：交易对手档案 / 客户经理岗
此类字段记录交易对手的身份及账户信息，由客户经理岗或清算岗从交易对手档案、清算记录中提取，用于追溯交易链路，完善交易场景。
XML 标签	字段内容	说明
TCNM	交易对手姓名 / 名称	字符型（128 位），个人对手填写姓名，机构对手填写全称，与对手身份文件一致，从交易对手信息记录中提取，避免简称或模糊表述。
TCIT	交易对手身份证件 / 证明文件类型	字符型（6 位），规则同 “客户身份证件类型（CITP）”，按 10.1 节代码表填写（如对手为个人填 110001，为机构填 610001），从对手身份资料中调取，明确对手身份类型。
TCID	交易对手身份证件 / 证明文件号码	字符型（128 位），规则同 “客户身份证件号码（CTID）”，居民身份证填 15/18 位，组织机构代码填 9 位，从对手身份认证记录中提取，确保对手身份可查。
TCAT	交易对手账户类型	字符型（6 位），规则同 “客户账户类型（CATP）”，按 10.2 节代码表填写（如对手个人账户填 120001，机构账户填 620001），从对手账户信息中调取，匹配对手账户属性。
TCAC	交易对手账号	字符型（64 位），填写交易对手的账户号码（如银行卡号、对公账号），从交易清算记录中提取，无对手账号时填 “@N”，确保交易对手账户可关联。
CFIN	对方金融机构网点名称	字符型（64 位），若交易对手账户在其他金融机构，填写该机构网点全称（如 “中国农业银行广州天河支行”），从清算机构信息中调取，无则填 “@N”。
CFIC	对方金融机构网点代码	字符型（16 位），填写对方网点的官方代码（如现代化支付系统行号），无代码时填 “@N”，从金融机构代码库中查询，与 CFIN 一一对应。
CFCT	对方金融机构网点代码类型	字符型（2 位），11 为现代化支付系统行号、12 为人民币结算账户管理系统行号、13 为银行内部机构号、14 为金融机构代码、99 为其他，与 CFIC 的代码类型匹配，无 CFIC 时填 “@N”。
CFRC	对方金融机构网点行政区划代码	字符型（9 位），规则同 “交易发生地（TRCD）”，填写对方网点所在地区的编码，无 CFIN 时填 “@N”，标识对手金融机构的地理位置。
RPMT	收付款方匹配号类型	字符型（2 位），按 10.5 节《收付款方匹配号类型代码表》填写（如 01 为大小额支付系统清算、02 为银联清算），从交易清算方式中判断，无匹配号时填 “@N”。
RPMN	收付款方匹配号	字符型（500 位），按对应匹配号类型规则填写（如银联清算填 “交易传输时间 + 系统跟踪号 + 受理机构码”），从清算匹配记录中提取，无匹配号时填 “@N”，用于关联收付款交易。
BPTC	银行与支付机构之间的业务交易编码	字符型（500 位），若交易涉及支付机构（如支付宝、微信支付），填写相关业务的交易编码，从支付业务清算记录中提取，无则填 “@N”，满足跨机构交易统计需求。
TSCT	涉外收支交易分类与代码	数值型（16 位），仅涉外交易填写，按 GB/T 19583-2014《涉外收支交易分类与代码》填写（如 121010 为 “货物贸易 - 一般货物”），从涉外交易申报记录中提取，无涉外交易时填 “@N”。
ROTF	交易信息备注	字符型（64 位），暂填写 “@N”，后续若有补充说明（如 “对手为关联企业”）可更新，由清算岗根据实际情况维护，补充交易细节。
六、代办人信息类（一级特征）—— 数据来源：代办业务登记 / 运营柜员岗
此类字段仅当交易存在代办人时填写，由运营柜员岗从代办业务登记单、身份核验记录中提取，无代办人则相关字段填 “@N”，确保代办交易可追溯。
XML 标签	字段内容	说明
TBNM	交易代办人姓名	字符型（128 位），填写代办人的真实姓名，与代办人身份证件一致，从代办业务登记单中提取，避免别名。
TBIT	交易代办人身份证件 / 证明文件类型	字符型（6 位），规则同 “客户身份证件类型（CITP）”，按 10.1 节代码表填写（如代办人为个人填 110001），从代办人身份资料中调取。
TBID	交易代办人身份证件 / 证明文件号码	字符型（128 位），规则同 “客户身份证件号码（CTID）”，居民身份证填 15/18 位，从代办人身份核验记录中提取，确保代办人身份真实。
TBNT	交易代办人国籍	字符型（3 位），规则同 “客户国籍（CTNT）”，按 GB/T 2659-2000 国别代码填写，从代办人身份信息中调取，明确代办人国籍。
七、人工补正与纠错标识类（一级特征）—— 数据来源：补正通知 / 反洗钱专职岗
此类字段用于人工补正或主动纠错场景，由反洗钱专职岗根据补正通知、纠错需求填写，确保报告信息可修正、可完善。
XML 标签	字段内容	说明
MIRS	人工补正标识	字符型（64 位），应答人工补正通知时填写补正通知文件名（如 “ABHC0000000000001-20240310-00000005”），其他情况填 “@N”，从补正通知文件中提取，标识补正关联关系。
OCNM	原客户号	字符型（32 位），仅修改 / 纠错报文填写，填写原大额交易报告中的客户号，从原报告中调取，确保修改对象精准。
OTDT	原大额交易发生日期	字符型（8 位），仅修改 / 纠错报文填写，填写原报告中的交易发生日期（格式同 HTDT），从原报告中调取，定位原交易时间。
OTCD	原大额交易特征代码	字符型（4 位），仅修改 / 纠错报文填写，填写原报告中的交易特征代码（格式同 CRCD），从原报告中调取，定位原交易特征。
OTIC	原业务标识号	字符型（256 位），仅修改 / 纠错报文填写，填写原报告中的业务标识号（格式同 TICD），从原报告中调取，定位原交易的唯一标识。
