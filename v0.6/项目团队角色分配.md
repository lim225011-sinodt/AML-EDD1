# AML-EDD1反洗钱项目团队角色分配

## 项目概述
- **项目名称**: AML-EDD1反洗钱增强尽职调查系统
- **项目版本**: v0.6开发阶段
- **项目目标**: 为农业银行等金融机构提供反洗钱合规检测和尽职调查服务
- **项目经理**: Claude Code

## 团队角色与职责分配

### 1. 业务经理 (Business Manager)
**角色定位**: 业务需求专家，负责银行业务需求和合规要求

**主要职责**:
- 负责农业银行等金融机构的业务需求收集和分析
- 确保系统功能符合2025年反洗钱10号令要求
- 提供300号文件接口格式的业务解读
- 协调与银行业务部门的沟通和需求确认
- 制定业务流程和操作规范

**具体工作内容**:
- 分析v0.6目录中的AML_BRD.pdf业务需求文档
- 提供反洗钱业务规则和风险评估标准
- 定义客户身份识别(KYC)和增强尽职调查(EDD)流程
- 制定可疑交易报告(STR)和合规检查标准
- 参与功能验收和业务测试

### 2. 产品经理 (Product Manager)
**角色定位**: 产品规划专家，负责产品功能规划和用户体验

**主要职责**:
- 制定产品功能规划和技术路线图
- 协调业务需求与技术实现的平衡
- 负责产品版本管理和发布计划
- 收集用户反馈并制定产品改进方案
- 确保产品符合市场需求和监管要求

**具体工作内容**:
- 基于业务需求制定v0.6版本功能清单
- 设计产品功能模块和用户操作流程
- 制定产品测试标准和验收标准
- 协调UI/UX设计与功能需求的匹配
- 负责产品文档编写和维护

### 3. 技术经理 (Technical Manager)
**角色定位**: 技术架构专家，负责技术实现和系统架构

**主要职责**:
- 设计系统技术架构和数据库结构
- 负责技术选型和开发框架搭建
- 制定代码规范和开发流程
- 解决技术难题和系统性能优化
- 确保系统安全性和数据保护

**具体工作内容**:
- 搭建Python 3.13开发环境和uv依赖管理
- 实现300号文件接口格式解析和对接
- 设计反洗钱检测算法和风险评估模型
- 构建存证链管理系统和数据存储方案
- 确保中文文本处理和UTF-8编码支持
- 负责技术团队的管理和代码审查

### 4. UI设计师 (UI Designer)
**角色定位**: 用户界面专家，负责系统界面设计和用户体验

**主要职责**:
- 设计系统用户界面和交互流程
- 确保界面符合金融行业专业要求
- 优化用户体验和操作便捷性
- 配合开发团队实现界面效果
- 维护设计规范和视觉标准

**具体工作内容**:
- 设计反洗钱检测界面的专业展示方式
- 设计客户尽职调查表单和数据录入界面
- 设计风险评分和预警结果的可视化展示
- 设计存证链管理和文档调阅界面
- 确保界面支持中文显示和专业术语
- 制作设计稿和交互原型

### 5. 测试人员 (QA Engineer)
**角色定位**: 质量保证专家，负责系统测试和质量控制

**主要职责**:
- 制定测试计划和测试用例
- 执行功能测试、性能测试和安全测试
- 编写自动化测试脚本和回归测试
- 跟踪缺陷管理和质量改进
- 确保系统符合业务需求和性能要求

**具体工作内容**:
- 编写基于脱敏数据的测试用例
- 执行反洗钱检测算法的准确性测试
- 测试300号文件接口的兼容性
- 进行系统性能和并发测试
- 执行安全性测试和渗透测试
- 建立持续集成和自动化测试流程

## 团队协作流程

### 1. 沟通机制
- **日常沟通**: 每日站会，汇报进展和问题
- **周例会**: 每周项目进展汇报和计划调整
- **月度回顾**: 月度项目总结和下月计划
- **专项会议**: 根据需要召开技术方案讨论会

### 2. 工作流程
1. **需求分析阶段**: 业务经理提供需求 → 产品经理制定方案 → 技术经理评估可行性
2. **设计阶段**: UI设计师设计界面 → 技术经理设计架构 → 产品经理确认方案
3. **开发阶段**: 技术经理组织开发 → 定期代码审查 → 持续集成测试
4. **测试阶段**: 测试人员执行测试 → 缺陷修复 → 回归测试
5. **验收阶段**: 业务经理验收 → 产品经理确认 → 项目经理总结

### 3. 文档管理
- 业务需求文档：业务经理负责
- 产品需求文档：产品经理负责
- 技术设计文档：技术经理负责
- 界面设计规范：UI设计师负责
- 测试计划和报告：测试人员负责

## 工作分配原则
1. **专业分工**: 各角色专注于自己的专业领域
2. **协作配合**: 跨角色协作确保项目质量
3. **责任到人**: 每项任务都有明确的负责人
4. **及时沟通**: 问题及时反馈和解决
5. **质量优先**: 确保交付质量和项目进度平衡
